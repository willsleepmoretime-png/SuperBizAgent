# Hybrid Weighted RRF：dev 参数实验 v1

## 实验约束

- 只使用 `rag-eval-v1` 的 `split=dev`。
- Dense candidates 固定 20，Final TopK 固定 5。
- test split 未用于选择参数。
- 每次完整 Case 结果保存在 `rag-eval-runs/`。

## 判断

H5 的 MRR 和 nDCG 最高，并且相对 Dense 有 3 条改善、1 条退化；但 Hit@3 下降 0.0286，MRR 只提高 0.0057，nDCG 只提高 0.0042，且本轮 P95 长尾明显。因此 H5 只能作为后续 Rerank 的候选输入配置，证据不足以直接冻结为默认最终排序。

H1 表明 BM25 candidates=5 会造成候选覆盖下降；不采用。H2 与 H3 质量完全相同，说明当前 Case 上 `rrfK=30/60` 未改变最终 Top5 排序。

下一步建议：使用 H5 产生较大的 Hybrid 候选集，不直接截断为最终答案；交给 `gte-rerank-v2` 重排后再评测 Hit@1、MRR、nDCG、Recall 与 P95。
